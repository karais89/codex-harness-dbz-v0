# Gameplay Loop

상태: 승인됨

## 목적

이 문서는 M1 First Playable Loop에서 사용할 `Start -> Move -> Pickup -> Deliver -> Result -> Retry`의 player-facing gameplay 기준을 정의한다.

이 문서는 2026-05-19에 사용자가 명시적으로 승인했다. Codex는 이 문서를 M1 First Playable Loop의 gameplay design 기준으로 사용할 수 있다.

이 문서는 구현 방식, C# 구조, Unity 씬 구성 세부를 정하지 않는다. M1 ExecPlan은 이 문서를 기준으로 플레이어가 직접 경험하는 규칙을 구현 방법과 검증 절차로 옮긴다.

## 적용 범위

이 문서는 M1의 단일 고정 레벨에서 플레이어가 보고 조작하는 핵심 루프를 다룬다.

- 시작 상태.
- 격자 기반 이동.
- 물건 획득.
- 목적지 배달.
- 성공 또는 실패 결과.
- Retry를 통한 재시작.
- 최소 UI 피드백.

## 제외 범위

이 문서는 다음을 M1 범위에서 제외한다.

- 자유 이동.
- 모바일 또는 터치 입력.
- 여러 레벨, 랜덤 배치, 레벨 선택.
- 스위치, 문, 밀기, 적, 위협 요소.
- 시간 압박.
- 점수 최적화.
- 서사 연출.
- 정확한 격자 크기, 좌표, 턴 수, 벽 위치의 design 고정.
- C# 스크립트 이름, private 메서드 구조, Unity 오브젝트 이름 같은 구현 세부.

## 결정 사항

- M1 이동은 격자 기반이다.
- 이동 입력 1회는 인접한 한 칸 이동 시도로 처리한다.
- 이동 입력은 방향키와 WASD를 지원한다.
- 벽 또는 격자 밖으로 이동하려는 입력은 이동하지 않고 턴도 소모하지 않는다.
- M1에는 단일 고정 레벨 1개만 포함한다.
- 단일 레벨은 작고 읽기 쉬운 직사각형 격자를 사용한다.
- 단일 레벨에는 경로 선택을 확인할 수 있는 2-3개의 막힌 칸 또는 벽을 둔다.
- 정확한 격자 크기, 좌표, 턴 수, 벽 위치는 이 문서에서 고정하지 않고 M1 ExecPlan 또는 구현 검증 단계에서 작고 읽기 쉬운 범위 안에서 정한다.
- 플레이어가 물건 칸에 올라가면 자동으로 Pickup된다.
- 플레이어가 물건을 가진 상태로 목적지 칸에 올라가면 자동으로 Deliver된다.
- M1에는 턴 제한 실패 상태를 포함한다.
- 턴 제한은 속도 경쟁이 아니라 짧은 퍼즐의 재시도 가능성을 검증하는 장치로 사용한다.
- 최소 UI에는 남은 턴, 물건 보유 여부, 결과 상태, Retry 버튼을 포함한다.
- 결과 상태는 진행 중, 성공, 실패를 구분한다.
- 성공 또는 실패 상태가 되면 이동 입력을 잠근다.
- Retry 버튼은 같은 레벨을 처음 상태로 되돌린다.

## 열린 질문

- 없음. 단, 정확한 격자 크기, 좌표, 턴 수, 벽 위치는 M1 ExecPlan 또는 구현 검증 단계에서 이 문서의 범위 안에서 결정한다.
