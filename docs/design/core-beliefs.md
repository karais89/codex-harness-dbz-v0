# Core Beliefs

상태: 작성 전

## 목적

이 파일은 아직 Delivery Bot Zero의 design 기준이 아니다.

`docs/design/core-beliefs.md`는 사용자 인터뷰와 공유된 이해 검증을 거친 뒤에만 `상태: 초안`으로 다시 작성할 수 있다. 현재 상태에서는 player-facing gameplay 설계, gameplay ExecPlan, gameplay 구현의 기준으로 사용할 수 없다.

사용자가 명시적으로 승인하기 전까지 Codex는 이 문서의 상태를 `상태: 승인됨`으로 바꾸지 않는다.

## 알려진 프로젝트 사실

- 이 저장소는 `Codex Harness v0 — Delivery Bot Zero`이다.
- 프로젝트는 작은 Unity 2D 퍼즐 게임을 사용해 Codex 주도 Unity 개발 워크플로우를 검증한다.
- M1 First Playable Loop는 아직 시작하지 않았다.
- `core-beliefs.md`만으로는 M1 gameplay ExecPlan을 시작할 수 없다.

## 작성 전 안내

- 이 파일의 현재 내용은 핵심 원칙이 아니다.
- Core Beliefs 초안은 사용자 인터뷰 후 공유된 이해 검증을 통과해야 작성할 수 있다.
- 인터뷰는 게임 정체성, 플레이어 경험 방향, 초기 마일스톤에서 하지 않을 것, 성공과 실패 판단 기준, Codex가 임의로 만들면 안 되는 design 결정을 다룬다.
- M1 세부 규칙, 좌표, 수치, 씬 구조, C# 스크립트 이름, 구현 방식은 이 문서의 작성 인터뷰에서 정하지 않는다.

## 승인 체크리스트

- [ ] 사용자 인터뷰를 완료했다.
- [ ] 공유된 이해 검증에서 사용자가 의도와 맞다고 확인했다.
- [ ] 인터뷰와 검증 내용을 바탕으로 이 문서를 `상태: 초안`으로 다시 작성했다.
- [ ] 사용자가 이 문서의 상태를 `승인됨`으로 바꾸라고 명시했다.
- [ ] M1 시작 전에 이 baseline이 프로젝트 방향 기준으로 충분한지 확인했다.
- [ ] M1에 필요한 조건부 design 문서가 있는지 별도로 판단했다.
